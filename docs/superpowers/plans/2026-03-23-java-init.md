# Java Init Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a minimal interactive Java init flow that generates `devenv` Java config for custom JDK selection plus Gradle and Maven toggles.

**Architecture:** Extend the existing enum-based language model and prompt-driven CLI flow, following the current Rust, Python, and Go implementation style. Reuse the existing Tera rendering pipeline so Java support fits into the same `ProjectContext -> templates -> files` path without new configuration layers.

**Tech Stack:** Rust, clap, dialoguer, serde, tera, cargo test
