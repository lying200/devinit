# Go Init Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a minimal interactive Go init flow that generates `devenv` Go config for version and package, including `go-overlay` handling when a version is chosen.

**Architecture:** Extend the existing language enum and prompt-based CLI flow in place, matching the established Rust and Python implementation style. Use the current Tera-based rendering pipeline for both `devenv.nix` and `devenv.yaml`, with `go-overlay` emitted only when Go version selection requires it.

**Tech Stack:** Rust, clap, dialoguer, serde, tera, cargo test
